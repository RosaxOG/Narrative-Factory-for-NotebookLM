# Security Policy

## Security Policy
We take security seriously at Narrative Factory for NotebookLM. This document outlines our security practices and guidelines around vulnerability reporting.

## Vulnerability Reporting Guidelines
If you discover a vulnerability in our project, please report it to us at our security email at security@narrativefactory.net. Please include a description of the vulnerability, steps to reproduce it, and any other relevant information.

## Supported Versions
We support the latest release of our product and the two previous versions. For more information on our supported versions, please refer to our [release notes](https://github.com/RosaxOG/Narrative-Factory-for-NotebookLM/releases).

## Security Best Practices  
1. Always keep your dependencies up to date.  
2. Use secure communication protocols.  
3. Regularly review your code for security vulnerabilities.  
4. Follow best practices for application security, as outlined by OWASP.  
5. Report any security vulnerabilities or potential threats immediately to ensure they are addressed.  
6. Educate yourself and your team about security awareness and practices.
